  • Week 1 Discussion

    Data Research/Project Intent

    Twitter can be used to simplify multiple logins in different apps with the “live” data resources. Therefore, a person will only need to remember their Twitter account and not several username and password combinations. Also, as Twitter has all personal information of the user, it can be used to quickly fill out that information on other forms including payments. Having all the information of a person stored within one general account like Twitter or Facebook makes it easier in using other outlets.

    The need for JSONP is because it allows for requesting data from a different domain. It seems that it is basically a code wrapper that retrieve code across different servers and gets around the same-origin policy.

    I can see that the geolocation works with Google Maps or MapQuest. Social networks work with contacts.

    When I sat down to think about where I wanted to go with the app, I may have come up with too much for just one month. The main focus for this project is retrieving vehicles information and using geolocation to display nearby repair facilities. Other features include using the camera as Barcode Scanner for the VIN of a vehicle. Also, I have seen examples of using the accelerometer for 0-60 times and lateral G’s. Using social networks for comparisons and questions or Instagram for photo sharing.

    Some past lessons learned for me is more of a mental note not to get frustrated when I get stuck. I used to spend too much time thinking about problems and end up over looking what usually turns out to be a simple syntax or even spelling error.

  • Week 2 Discussion

    Accessibility

    I think the best way to make an app more accessible is through clean design and user-friendly objectives. The HTML code needs to be semantic and tags should be used appropriately. With using jQuery Mobile, each page should have elements that have tags fitting to that element and be consistent through other pages. Re-using the same tag like div or section within the same page should be limited.

    The most difficult part I can see with screen readers is the positioning of the content. Unfortunately, everything isn’t designed the same and content can be formatted different ways. However, using semantic code can help define elements like titles and content with a corresponding tag. This allows the reader to better explain the content to the user.

    I see that the iPhone is one of the leading devices for seeing impaired people. The integration of Siri is not just for easy use but allows blind people to utilize modern technology to make their lives easier. Project Ray is another device that plans to create smartphone geared directly to seeing impaired consumers. They joined together with Qualcomm, which is the largest mobile technology company, to create a completely new user interface that assist the blind and will actually learn for usage patterns.

    Shine Reader is a great app for accessibility for blind or seeing impaired users. It creates a traditional keypad overlay over the screen to allow easy input. The entire screen is turned into a grid of buttons that makes for larger button surface. Also it contains speech recognition to tell the user the input and content. Georgie is another amazing app for the blind, which has an interface that responds and relays info to the user. Everything is customizable for the user down to the gps telling the user about their surroundings.
